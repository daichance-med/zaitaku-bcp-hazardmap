【このアプリについて】
災害時の患者安否確認を効率化する無料ツールです。
患者住所を地図上に自動表示し、洪水・土砂災害・津波などの
ハザード情報と組み合わせて可視化します。

在宅医療（訪問診療・訪問看護・訪問介護・訪問薬局など）に
携わる全ての事業所でご利用いただけます。

【患者リストの作成方法】

患者リストの作成には2つの方法があります：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【方法1】テンプレートに直接入力（シンプル）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 「患者リストテンプレート_v2.2.xlsx」を開く
2. 「患者リスト」シートに患者情報を入力
3. Excel形式（.xlsx）で保存
4. アプリにアップロード

必須項目：
- 患者番号 ......... 管理番号（重複不可）
- 患者氏名 ......... フルネーム
- 住所 ............. 都道府県から記入（例：神奈川県小田原市○○1-2-3）

任意項目：
- 自宅電話番号 ..... 固定電話（市外局番なしでOK、自動補完されます）
- その他の電話番号 . 携帯電話や連絡先（複数ある場合は「/」で区切る）
                     例：090-1234-5678 / 03-9999-8888
- リスク分類 ....... 高/中/低（色分けに使用）

※座標（lat/lon）列は空欄でOK。住所から自動計算されます。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【方法2】電カルのCSVデータを貼り付け（効率的）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

電子カルテからエクスポートしたCSVデータをテンプレートに
貼り付けることで、効率的に患者リストを作成できます。

手順：

ステップ1: 電カルからCSVエクスポート
- 患者一覧をCSV形式でエクスポート
- 患者番号、氏名、住所、電話番号などを含むデータ

ステップ2: CSVをExcelで開く
- エクスポートしたCSVファイルをダブルクリック

ステップ3: 必要な列をコピー
- 患者番号、氏名、住所、電話番号などの列を選択
- Ctrl+C でコピー

ステップ4: テンプレートの該当列に貼り付け
- 「患者リストテンプレート_v2.2.xlsx」を開く
- 「患者リスト」シートの該当列を選択
- Ctrl+V で貼り付け

⚠️ 重要：住所が分割されている場合

電カルによっては、住所が複数の列に分かれている場合があります：
例）D列：都道府県、E列：市区町村、F列：番地

【統合方法】
1. 空いている列（例：J列）に統合用の数式を入力
   J2セルに：=D2&E2&F2
2. Enter を押す
3. J2セルの右下をダブルクリックして全行にコピー
4. J列全体を選択して Ctrl+C（コピー）
5. J列を右クリック → 「形式を選択して貼り付け」→「値」→ OK
6. J列をコピーして、テンプレートの「住所」列に貼り付け

⚠️ 超重要：必ず「値」として貼り付けること！

数式のままコピーすると、テンプレートでエラーになります。
必ず「値」に変換してから貼り付けてください。

【確認方法】
- セルをクリックして、上部の数式バーを確認
- 数式（=D2&E2&F2）と表示される → NG
- 文字列（東京都新宿区...）と表示される → OK

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【CSV直接アップロード（上級者向け）】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

以下の条件を満たすCSVファイルは、直接アップロードできます：
✓ 住所が1列にまとまっている
✓ 電話番号が2列以内
✓ 列名が標準的（患者番号、患者氏名、住所など）

アップロード後、列マッピング画面が表示されます。
CSVの列とアプリの項目を対応付けてください。

【電話番号の入力について】
例：小田原市の場合
- 入力: 48-5409 → 自動で「0465-48-5409」に変換されます
- 携帯: 090-1234-5678 → そのまま使用されます

市外局番の自動補完に対応している主要都市：
東京23区、横浜市、川崎市、大阪市、名古屋市、福岡市、
札幌市、仙台市、広島市、京都市、神戸市、小田原市 など

【トラブルシューティング】

Q. 地図が表示されません
A. インターネット接続を確認してください。
   ブラウザのキャッシュをクリア（Ctrl+Shift+Delete）してみてください。

Q. Excelファイルがアップロードできません
A. ファイル形式が.xlsx形式であることを確認してください。
   古い.xls形式は非対応です。

Q. 座標が正しく表示されません
A. 住所が都道府県から正しく入力されているか確認してください。
   例：○「神奈川県小田原市前川367-1」
       ×「小田原市前川367-1」

Q. 電話番号に市外局番がつきません
A. 住所に都道府県名が含まれているか確認してください。
   対応していない市区町村の場合は、電話番号に直接市外局番を
   含めて入力してください（例：0465-48-5409）。

Q. スマホで使えますか？
A. はい、使えます。HTMLファイルをスマホに転送し、
   ブラウザで開いてください。タッチ操作に最適化されています。

Q. CSVから貼り付けたら #REF! エラーが出ます
A. 数式のまま貼り付けている可能性があります。
   必ず「値」として貼り付けてください。
   手順：コピー → 右クリック → 形式を選択して貼り付け → 値

Q. 住所が複数列に分かれています
A. Excel上で統合してから貼り付けてください。
   例：J2セルに =D2&E2&F2 と入力 → 全行にコピー
   → J列を「値」に変換 → テンプレートに貼り付け

Q. 郵便番号は必要ですか？
A. 不要です。住所（都道府県から番地まで）だけで
   正確に座標変換できます。

Q. CSV形式で保存してもいいですか？
A. Excel形式（.xlsx）を推奨します。
   CSV形式だと書式や数式が失われます。

【セキュリティに関する重要な注意事項】
========================================

このアプリは患者の個人情報を取り扱います。
以下のセキュリティ対策を必ず実施してください。

【データの取り扱い】
• 本アプリはブラウザ上で完結し、データを外部サーバーに送信しません
• 地図作成後、元データ（jsonData等）は自動的に削除されます
• ただし、アップロードしたファイル自体はPCに残ります
• 使用後は必ずファイルを削除してください

【推奨するセキュリティ対策】
1. 暗号化USBメモリの使用
   • パスワード保護機能付きUSBを使用
   • 推奨製品：BUFFALO RUF3-HSシリーズ等

2. 電子カルテ端末での直接使用
   • データを外部に持ち出さない
   • 同一端末で完結させる

3. 使用後の確実な削除
   • ファイルを削除
   • ゴミ箱を空にする
   • ブラウザを完全に閉じる

4. 共有PC・公共PCでの使用禁止
   • 必ず管理された端末で使用
   • 個人PCの場合はパスワード保護必須

【万が一、情報漏洩が発生した場合】
1. 直ちに上司・管理者に報告
2. 個人情報保護委員会への報告を検討
3. 影響を受ける患者への通知を検討
4. 再発防止策の実施

### CSV機能（電子カルテ連携）

電子カルテから出力したCSVファイルを直接読み込めます。

【使い方】
1. 電子カルテで患者リストをCSV形式でエクスポート
2. アプリにCSVファイルをドロップ
3. 列の対応付け画面が表示される
4. CSVの列名とアプリの項目を対応付ける
5. 「変換実行」をクリック
6. 自動で地図作成

【メリット】
• 手入力不要！
• 作業時間が95%削減（100名で2-3時間 → 5分）
• 入力ミスがゼロ

---

### JSON機能（BCP地図の保存）⭐重要

作成した地図を保存して、後から即座に呼び出せます。

【使い方】

＜地図の保存＞
1. 地図作成後、右下の「💾 地図データ保存」をクリック
2. 「BCP地図_○○_2026-02-09.json」というファイルがダウンロードされる
3. このファイルを安全な場所に保存

＜地図の読み込み＞
1. 「在宅BCPハザードマップアプリ.html」を開く
   （ダブルクリックでブラウザで開く）
2. ファイルアップロード画面が表示される
3. 保存しておいた「BCP地図_○○.json」をドロップ
4. 3秒で地図が表示される！

⚠️ 重要な注意点
• JSONファイルを直接開いても地図は表示されません
• JSONファイルは「データ」なので、文字列が表示されるだけです
• 必ず「在宅BCPハザードマップアプリ.html」を開いてから、
  JSONファイルをドロップしてください

【2つのファイルの関係】
• 在宅BCPハザードマップアプリ.html = アプリ本体（地図を表示するソフト）
• BCP地図_○○.json = データ（患者情報）

→ アプリにデータを読み込ませて使います

【推奨フォルダ構成】
BCP資料/
├── 在宅BCPハザードマップアプリ.html  ← アプリ本体
├── BCP地図_2026-02-09.json           ← データ（最新）
└── BCP地図_2026-01-15.json           ← データ（前月・バックアップ）

【メリット】
• 座標変換が不要（超高速）
• 完全な機能を維持（電話発信、ハザード表示）
• ファイルサイズが軽い（数十KB）
• セキュリティ向上（元のExcelは削除できる）

【推奨運用】
1. 月1回、最新Excelで地図を作成
2. JSONで保存
3. 元のExcelは削除
4. 災害時はJSONを読み込んで即使用
